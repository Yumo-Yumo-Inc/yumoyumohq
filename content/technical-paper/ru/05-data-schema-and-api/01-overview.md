# 05 — Схема данных и API

В данном разделе определяется публичная схема и API-интерфейс Yumo Yumo. Выходные данные конвейера в 02, решения уровня доверия в 03 и учёт вознаграждений в 04 отображаются на формы сущностей и событий, определённые здесь.

Схемы публикуются, чтобы участники протокола могли читать одну и ту же запись одинаковым образом. Физическая стратегия индексирования, выбор провайдера, перемещение горячих/холодных данных и коммерческие параметры B2B-продукта остаются в операционной документации.

## 5.0 Принцип схемы

| Принцип | Следствие |
|---|---|
| Типизированная запись | Каждый публичный объект версионируется и привязан к схеме |
| Событийный реестр | Учёт вознаграждений выводится из поддающихся аудиту исторических событий |
| Разделённые данные | Персональные данные, агрегированные данные и сводка в блокчейне находятся в отдельных слоях |
| Версионируемый API | Добавление/удаление полей следует правилам обратной совместимости |
