# Утилитарность INT

## 4.27 Утилитарная поверхность

INT выполняет шесть функций внутри протокола Yumo Yumo. Две из них активны; четыре запланированы и активируются по мере созревания поверхности протокола.

| Функция | Статус | Описание |
|---|---|---|
| **Актив вознаграждения** | Активна | Пользователи зарабатывают INT через жизненный цикл конверсии bINT → INT (4.4) за верифицированный вклад Proof of Expense |
| **Актив стейкинга** | Активна при TGE+1 неделя | Держатели блокируют INT в уровневых стейкинговых пулах (4.6) для получения вознаграждений из стейкингового распределения |
| **Цель выкупа-и-сжигания** | Активна при начале выручки | Казначейство использует выручку от продукта данных для покупки INT на открытом рынке и сжигания (4.9), создавая дефляционное давление |
| **Сжигание за отчёт данных** | Запланирована | Бизнесы, получающие агрегированные отчёты данных сообщества, обязаны сжечь определённое количество INT за каждый отчёт (4.30) |
| **Сигнал управления** | Запланирована | Взвешенное по INT сигнализирование для решений по приоритетам продукта данных, распределению казначейства и грантам экосистемы |
| **Связанный API-доступ** | Запланирована | Потребители API анонимизированного продукта данных могут быть обязаны блокировать INT в качестве обеспечения своих ключей доступа |

## 4.28 Реальный доход

INT спроектирован как актив реального дохода. Его долгосрочное накопление стоимости обеспечивается внешней выручкой, генерируемой бизнесом продукта данных, а не эмиссией токенов.

Два механизма связывают выручку платформы с держателями токенов:

1. **Доход стейкеров.** Часть чистой выручки от продукта данных поступает в пул доходности стейкеров, доступный INT-стейкерам пропорционально их доле и весу уровня.
2. **Выкуп-и-сжигание.** Часть чистой выручки используется казначейством для покупки INT на открытом рынке и бессрочного сжигания.

Распределение между доходностью стейкеров и выкупом-и-сжиганием является параметром казначейской политики, управляемым в рамках контролей, описанных в 4.10. Оба потока привязаны к внешней выручке, а не к эмиссии токенов. Это разделение означает, что эмиссия (4.19) предсказуемо расширяет предложение, тогда как реальный доход сжимает или распределяет предложение на основе фактических результатов бизнеса.

## 4.29 Источники выручки

Выручка, питающая механизм реального дохода, поступает из:

- **Продажи анонимизированных данных.** k-анонимизированные, агрегированные данные на уровне чеков, продаваемые FMCG-брендам, розничным сетям, исследовательским фирмам и разработчикам через уровневый API-доступ.
- **Партнёрская и реферальная выручка.** Переходы по сравнению цен к розничным или купонным партнёрам (запланировано).
- **Премиум-подписка.** Расширенная персональная аналитика и функции автоматизации целей (запланировано).

Детали генерации выручки и архитектура анонимизации описаны в 05 Схема данных и API.

## 4.30 Сжигание за отчёт данных

Бизнесы, приобретающие агрегированные отчёты данных сообщества, обязаны сжечь определённое количество INT за каждый генерируемый отчёт. Сжигание происходит on-chain и является бессрочным.

Этот механизм выполняет две задачи:

1. **Дефляционное давление.** Каждый потреблённый отчёт данных безвозвратно выводит INT из оборотного предложения, создавая дефицит на стороне спроса, пропорциональный коммерческому принятию продукта данных.
2. **Согласование ценности.** Требование сжигания напрямую связывает утилитарность продукта данных с токеном. По мере того как всё больше бизнесов потребляет данные Yumo Yumo, всё больше INT выводится из обращения, укрепляя связь между использованием платформы и ценностью токена.

Объём сжигания за отчёт устанавливается казначейской политикой и варьируется по уровню отчёта (например, агрегаты на уровне категорий и полные панели на уровне корзины). Ценовая структура гарантирует, что стоимость сжигания остаётся малой долей коммерческой ценности продукта данных, при этом генерируя значимый кумулятивный эффект на предложение.
