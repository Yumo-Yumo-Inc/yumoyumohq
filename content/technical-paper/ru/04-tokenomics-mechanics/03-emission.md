# Эмиссия вознаграждений пользователям

## 4.3 Как вознаграждения поступают в bINT

Пул вознаграждений пользователей (User Rewards) управляется таблицей распределения из 4.17. Внутри этого пула ежедневная эмиссия измеряется кривой, основным входом которой является ежемесячная активная аудитория (MAU). У кривой есть три свойства, достойных упоминания:

- **Пошаговый рост к пику.** По мере роста MAU через определённые диапазоны ежедневный эмиссионный пул расширяется ступенями, а не непрерывно. Это позволяет избежать эффекта обрыва, когда активность колеблется вблизи границы.
- **Ограниченный пик.** Ежедневный пул растёт ступенчато до пикового диапазона, затем удерживается. После пика дополнительный MAU повышает плотность вклада на пользователя. Значения диапазонов калибруются в продакшене и не публикуются.
- **Долгий горизонт.** Рельс вознаграждений пользователей рассчитан на 15-летний горизонт. Доля предложения, выделенная под вознаграждения (64,35 млрд INT, см. 4.17), является бюджетом; кривая — счётчиком.

Ступенчатая функция — диапазоны MAU, значения ежедневного пула для каждого диапазона и переходное поведение — задокументирована в 4.19. Границы диапазонов перенастраиваются по мере развития наблюдаемой активности.

## 4.4 Жизненный цикл конверсии bINT → INT

bINT начисляется вне блокчейна в момент, когда чек проходит уровень доверия (03). Он рассчитывается в INT через периодическую (еженедельную) эпоху, а не через пользовательский on-chain-вызов конверсии. Жизненный цикл:

```
начисление  →  удержание  →  расчёт (эпоха)  →  получение  →  INT в кошельке пользователя
```

- **Начисление (accrue).** За каждый чек, в учётном слое вне блокчейна. Сумма определяется полосой доверия, ежедневным потолком пользователя и текущей эмиссионной ступенью.
- **Удержание (hold).** bINT остаётся в учётном слое в течение минимального окна удержания до получения права на расчёт. Это окно даёт уровню доверия (03) время отреагировать на аномальные паттерны до того, как будет распределён какой-либо INT.
- **Расчёт (settle).** Каждую эпоху пригодный bINT конвертируется в INT по фиксированному коэффициенту 1:1 (4.24). Движок строит список распределения, независимый верификатор проверяет его (4.17), а полученный корень публикуется в аудированный дистрибьютор.
- **Получение (claim).** Пользователь забирает свой INT напрямую из дистрибьютора в стандартный SPL-кошелёк, передаваемым. Казначейство удерживает INT до получения; отдельного шага вестинга нет.

Когда суммарное пригодное вознаграждение эпохи превышает глобальный потолок эмиссии, сумма каждого участника пропорционально снижается на один и тот же коэффициент (пропорциональное снижение по мягкому потолку, soft-cap pro-rata, 4.24). Длина окна удержания и значение глобального потолка управляются на операционном уровне и не публикуются.

## 4.5 Ежедневный потолок в терминах токеномики

Эффективный ежедневный потолок bINT является произведением базового потолка, множителя уровня (03 §3.6) и текущего здоровья пользователя (03 §3.5). Текущая реализация MVP использует таблицы по уровням (4.22); целевая архитектура использует потолок на основе формулы (4.23). Множитель и значения health специфичны для пользователя и определяются на уровне доверия.

Это разложение важно, потому что оно позволяет протоколу перенастраивать любой из трёх факторов, сохраняя токеномику. Расширение рынка может повысить базу; перебалансировка системы уровней может сдвинуть множитель; волна злоупотреблений может сжать распределение health.
