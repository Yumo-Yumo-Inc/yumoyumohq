# Классы токенов

## 4.1 Четыре класса

Yumo Yumo работает с четырьмя классами активов, у каждого своя отдельная роль. Только два из них — on-chain-токены; остальные два — записи вне блокчейна.

| Класс | Форма | Модель передачи | Роль |
|---|---|---|---|
| **INT** | SPL-токен на Solana | Передаваемый на рынке | Координация на уровне протокола, стейкинг, экосистемные стимулы. Параметры предложения изложены в Vision Paper. |
| **bINT** | Учётная единица вне блокчейна (операционный уровень) | Рассчитывается в INT через определённый жизненный цикл | Учётный слой вклада между работой и вознаграждением. |
| **ePoints** | Запись вне блокчейна, деноминированная в USD | Внутрипродуктовый кредит инсайтов | Запись скрытых издержек на уровне домохозяйства, выявленных по каждому верифицированному чеку. |
| **Foundation NFT (Yumbie)** | Непередаваемый актив Token-2022 | Непередаваемый | Постоянная идентичность. Эволюционирует в Smart Agent на вехе, определённой в Vision Paper. |

### Зачем четыре класса

Vision Paper объясняет причину с точки зрения пользовательского опыта. Причина с точки зрения механики — разделение ответственности:

- INT перемещается по рынкам и биржам; он передаваемый и взаимозаменяемый.
- bINT измеряет вклад и рассчитывается в INT; это единица вне блокчейна, поэтому учёт может развиваться без on-chain-миграции.
- ePoints несут сигнал экономического инсайта в качестве собственной записи вне блокчейна, поэтому пользовательская аналитика может расти, в то время как предложение INT остаётся фиксированным.
- Foundation NFT обеспечивает непрерывность идентичности в качестве непередаваемого актива Token-2022, по одному на кошелёк.

## 4.2 Структура полномочий

Полномочия различаются в зависимости от того, является ли класс on-chain или off-chain.

- **Полномочие на выпуск INT (mint authority)** — удерживается только до момента, когда всё предложение выпущено на генезисе, затем закрывается. После генезиса INT выпустить нельзя; распределение — это казначейский перевод через аудированный дистрибьютор (4.15).
- **Казначейство и сжигание INT** — удерживаются мультиподписью Squads, с раздельными согласованиями для подписания корня распределения, движения казначейства и clawback резерва.
- **bINT и ePoints** — учётные единицы вне блокчейна на операционном уровне. У них нет on-chain-полномочий на выпуск или заморозку; их балансы рассчитываются в INT через жизненный цикл из 4.4.
- **Foundation NFT** — Token-2022 с расширением непередаваемости, выпускается бэкендом один раз на кошелёк. Непередаваемость обеспечивается на уровне токен-программы.

Хранение bINT и ePoints вне блокчейна убирает пособытийные on-chain-полномочия из пути вклада; единственное полномочие уровня INT, сохраняющееся после генезиса, — это мультиподпись над казначейством, корнями распределения и сжиганиями.
