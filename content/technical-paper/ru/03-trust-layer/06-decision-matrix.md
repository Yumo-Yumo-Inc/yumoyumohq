# Матрица решений

## 3.11 От оценки к результату

Как только чек получил полосу доверия, а пользователь — актуальный снимок здоровья, уровень направляет чек к одному из четырёх результатов.

| Результат | Что видит пользователь | Что записывает реестр |
|---|---|---|
| **Принятие — полный кредит** | Подтверждённое превью + полная сумма bINT для этого чека и этого пользователя. | `receipt.status = "verified"`, полный кредит, перечислены семейства сигналов. |
| **Принятие — сниженный кредит** | Подтверждённое превью + меньшая сумма bINT. Без дополнительного трения. | `receipt.status = "verified"`, частичный кредит, категория причины понижения. |
| **Удержание для рассмотрения** | "Мы проверяем этот чек. Результат обычно приходит в течение дня." | `receipt.status = "under_review"`, запись попадает в очередь апелляций (3.12). |
| **Отклонение** | Чёткое сообщение на простом языке и 0 bINT. | `receipt.status = "rejected"`, категория причины отклонения. |

Четвёртый результат зарезервирован для случаев за пределами полосы правдоподобности честного чека — например, изображение, отмеченное проверками аутентичности синтетических медиа, рукописный документ или дубликат чека, уже зачисленного другому пользователю с конфликтующими доказательствами.

## 3.12 Очередь апелляций

Чек, удержанный для рассмотрения, попадает в очередь с целевым операционным сроком обработки. Рецензент (изначально операционная команда, позже — пул сообщества, зарабатывающий Proof of Contribution) видит:

- Изображение чека и извлечённую запись.
- Полосу и список семейств сигналов, повлиявших на решение.
- Быстрый обзор недавней истории пользователя.
- Три действия: **одобрить полностью**, **одобрить частично**, **оставить отклонение в силе**.

Рецензент видит те же семейства сигналов, что записаны в блоке чека. Это сохраняет согласованность с логикой уровня и поддерживает последовательные решения.

Если рецензент отменяет рекомендацию уровня, отмена записывается и вносит вклад в следующий цикл калибровки.

## 3.13 Что может сделать пользователь

Пользователь, чей чек отклонён, видит объяснение на уровне категории и, где уместно, путь самообслуживания: переснять чек при лучшем освещении, связаться со службой поддержки с подтверждением оплаты или принять отклонение. Причины на уровне сигналов остаются во внутренней конфигурации доверия.

Пользователь, чей показатель здоровья был снижен, может восстановить его, внося чистые чеки. Восстановление намеренно постепенное; система вознаграждает устойчивое хорошее поведение, а не кратковременные всплески.
