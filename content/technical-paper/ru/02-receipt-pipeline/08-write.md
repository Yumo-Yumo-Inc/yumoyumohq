# Этап 6 — Запись

## 2.9 Этап 6 — Запись выходных данных

Одна транзакция Postgres записывает:

```
INSERT INTO receipts (...) VALUES (...);
INSERT INTO receipt_line_items (...) VALUES (...);
INSERT INTO price_observations (...) VALUES (...);
INSERT INTO events (event_type, payload) VALUES ('receipt.verified', {...});
```

Строка `events` запускает двух нисходящих потребителей:

- **Скорер доверия** (03) — получает событие, вычисляет оценку доверия, записывает в `trust_scores`.
- **Рабочий расчётов** — ставит в очередь кредит `bINT.pending`. Фактический ончейн-минт происходит в асинхронном уровне (01 Фаза B).

Транзакция идемпотентна по внутреннему ключу записи — безопасна для повторного воспроизведения, если рабочий повторяет попытку.
