# Этап 2 — Структурированное извлечение

## 2.5 Этап 2 — Структурированное извлечение

На этом этапе выход слоя чтения документа преобразуется в объект `ReceiptExtraction`. Публичный контракт — это схема и поведение этапа; провайдеры моделей, текст промптов, политика маршрутизации, бюджеты токенов и условия повторных попыток управляются внутренним операционным слоем.

### Граница маршрутизации моделей

Yumo Yumo выполняет структурированное извлечение за модельно-независимым интерфейсом. Операционная политика может выбирать подходящий движок на основе языка, сложности документа, состояния здоровья и сигналов качества. Веса, порядок и резервное поведение этой политики остаются приватными.

### Структурированный результат

```json
// ReceiptExtraction
{
  "merchant": {
    "name_raw": "MIGROS T.A.S.",
    "tax_id_raw": "1234567890",
    "address_raw": "...",
    "phone_raw": null
  },
  "captured_at_raw": "17/05/2026 14:23",
  "currency": "TRY",
  "totals": {
    "subtotal": 234.50,
    "tax_total": 42.21,
    "grand_total": 276.71
  },
  "tax_lines": [
    { "rate_pct": 18.0, "base": 200.0, "amount": 36.0 },
    { "rate_pct": 8.0, "base": 77.50, "amount": 6.20 }
  ],
  "payment_method": "credit_card",
  "line_items": [
    {
      "raw_text": "SUT 1L PINAR",
      "qty": 2,
      "unit_price": 23.50,
      "line_total": 47.00,
      "tax_rate_pct": 8.0
    }
  ],
  "quality_band": "medium",
  "extraction_notes": "tax_total reconstructed from tax_lines"
}
```

Схема сводит выход модели к форме, которую может проверить слой правил. Итоги, дата, валюта, строковые позиции и налоговые поля проверяются повторно на следующем этапе.

### Обработка согласованности

Если извлечение получает низкую оценку качества или слой правил обнаруживает несоответствие, конвейер может отправить результат на проверку или повторную обработку. Выбор пути управляется операционными параметрами.
