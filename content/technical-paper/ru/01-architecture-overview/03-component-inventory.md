# Перечень компонентов

## 1.2 Перечень компонентов

Данный перечень описывает ответственность компонентов протокола и контракты, которые они предоставляют.

| Компонент | Ответственность | Публичный контракт |
|---|---|---|
| Клиентское приложение | Захват чека, подпись кошелька, предварительный просмотр пользователя | Подпись пользователя выполняется на устройстве; загрузка начинается явным действием пользователя |
| API-интерфейс | Идентификация, оркестрация загрузки, запуск конвейера, запросы статуса | Стабильный REST/SDK интерфейс; статусы этапов и категории ошибок |
| Конвейер обработки чеков | Преобразование входных изображений/PDF в структурированную запись чека | Типизированные промежуточные выходные данные, состояние проверки, канонические ссылки на продукт и торговца |
| Уровень доверия | Формирование права на вознаграждение на основе чека и сигналов пользователя | Публичные диапазоны, категории решений и калибровка, управляемая во внутренних операциях |
| Реестр и учёт вознаграждений | Хранение событий bINT/ePoints в виде неизменяемого потока бухгалтерских записей | Модель событий только с добавлением, поддающиеся аудиту записи расчётов |
| Слой данных | Формирование анонимизированных агрегатов на основе наблюдений за чеками | Разделение персональных данных, k-пороги, версионированные агрегированные выходные данные |
| Программы в блокчейне | Состояние токенов, стейкинг, маршрутизация казначейства и криптографические обязательства | Поддающиеся аудиту интерфейсы программ и опубликованные адреса программ |
| Операционная плоскость управления | Мониторинг, квоты, маршрутизация поставщиков, реагирование на инциденты | Публичная сводка статуса; runbook, пороги и детали маршрутизации остаются приватными |

Это разделение важно для безопасности: публичный документ делает архитектуру поддающейся аудиту, а комбинации инфраструктуры, пороги и шаги реагирования управляются во внутренних операциях.
