# Схема системы высокого уровня

## 1.1 Схема системы высокого уровня

```mermaid
flowchart LR
    subgraph Client["Устройство пользователя"]
        A["Приложение<br/>подпись кошелька + захват чека"]
    end

    subgraph Processing["Синхронная обработка"]
        B["API-интерфейс"]
        C["Конвейер обработки чеков"]
        D["Уровень доверия"]
    end

    subgraph Data["Данные вне блокчейна"]
        E[("Записи чеков")]
        F[("Реестр bINT / ePoints")]
        G["Анонимизированные агрегаты"]
    end

    subgraph Chain["Уровень в блокчейне"]
        H["Программы токенов"]
        I["Казначейство и стейкинг"]
        J["Криптографические обязательства"]
    end

    A --> B --> C --> D
    D --> E
    D --> F
    E --> G
    F -. "пакетное урегулирование" .-> H
    F -. "обязательство" .-> J
    H --> I
```

Схема показывает границу публичной архитектуры: пользовательский предпросмотр является синхронным; учёт bINT и ePoints сначала записывается в реестр, а затем переносится на уровень в блокчейне расчётными workers. Диаграмма сфокусирована на компонентах протокола и потоках данных.
