# Допущения масштабируемости

## 1.8 Допущения масштабируемости

Этот раздел выражает ёмкость через переменные рабочей нагрузки. Архитектура совместно оценивает число пользователей, плотность чеков, количество строк, частоту повторных попыток и политику пакетной обработки.

| Символ | Значение |
|---|---|
| `U` | Ежедневные активные пользователи |
| `r` | Среднее количество чеков на пользователя в день |
| `a` | Доля чеков, отправленных на обработку |
| `L` | Среднее количество строк в чеке |
| `v` | Доля проверенных чеков, имеющих право на вознаграждение |
| `e` | Среднее количество записей событий на чек |
| `ρ_ocr` | Частота повторных попыток OCR |
| `ρ_llm` | Частота повторных попыток или самосогласования LLM |
| `B` | Размер пакета для расчётов в блокчейне |

Ежедневный объём обработанных чеков:

```text
R_d = U × r × a
```

Приблизительный ежемесячный рост горячих данных:

```text
Rows_m ≈ 30 × R_d × (1 + L + e)
```

Ежедневный объём вызовов моделей:

```text
OCR_d ≈ R_d × (1 + ρ_ocr)
LLM_d ≈ R_d × (1 + ρ_llm)
```

Ежедневный объём вознаграждений и расчётов:

```text
Verified_d ≈ R_d × v
Onchain_batches_d ≈ ceil(Verified_d / B)
```

Публичная форма переменных затрат:

```text
Cost_d ≈ OCR_d × c_ocr + LLM_d × c_llm + Storage_d × c_storage + Settlement_d × c_chain
```

Решения о ёмкости принимаются на основе совместного измерения `U`, `r`, `L`, частот повторных попыток и политики пакетной обработки. Числовые пороги и коэффициенты затрат, специфичные для поставщиков, остаются в операционном планировании.
