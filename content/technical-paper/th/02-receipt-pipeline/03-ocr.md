# ขั้นตอนที่ 1 — การอ่านเอกสาร

## 2.4 ขั้นตอนที่ 1 — ชั้นการอ่านเอกสาร

ขั้นตอนนี้แยกบล็อกข้อความ ลำดับการอ่าน และข้อมูลตำแหน่งจากภาพใบเสร็จหรือข้อมูลเข้า PDF สัญญาสาธารณะคือผลลัพธ์ที่ปรับมาตรฐานแล้วของขั้นตอน ไม่ใช่ชื่อผู้ให้บริการ

### การปรับมาตรฐานผลลัพธ์

เครื่องมืออ่านเอกสารสามารถส่งคืนโครงสร้างที่แตกต่างกัน ไพพ์ไลน์จะปรับมาตรฐานให้อยู่ในรูปแบบภายในเดียว:

```json
// DocumentReadResult
{
  "raw_text": "MIGROS\nFIS NO: 4521\n...",
  "blocks": [
    {
      "text": "MIGROS",
      "bbox": { "x": 120, "y": 40, "w": 200, "h": 50 },
      "confidence_band": "high",
      "reading_order": 0
    }
  ],
  "quality_band": "high",
  "detected_languages": ["tr"],
  "page_count": 1
}
```

บล็อกถูกจัดเรียงตามลำดับการอ่านและส่งต่อไปยังขั้นตอนถัดไปเป็นข้อมูลเข้าแบบกำหนดได้ ดังนั้นการแยกข้อมูลของโมเดลจึงไม่ผูกติดกับรูปแบบการตอบสนองดิบของผู้ให้บริการใด ๆ

### สัญญาณคุณภาพ

ขั้นตอนการอ่านเอกสารนำแบนด์คุณภาพและหมวดหมู่ข้อผิดพลาดไปสู่ขั้นตอนถัดไป ในกรณีที่มีคุณภาพต่ำ ไพพ์ไลน์สามารถประมวลผลใหม่ ขอภาพใหม่จากผู้ใช้ หรือดำเนินการต่อด้วยความเชื่อมั่นต่ำลงตามนโยบายปฏิบัติการ

สิ่งนี้รักษาสัญญาทางเทคนิคสาธารณะไว้ ในขณะที่หลีกเลี่ยงรายละเอียดเกี่ยวกับค่าเกณฑ์และพฤติกรรมสำรองที่อาจถูก reverse engineer ได้ง่าย
