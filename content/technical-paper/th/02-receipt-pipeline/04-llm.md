# ขั้นตอนที่ 2 — การแยกข้อมูลเชิงโครงสร้าง

## 2.5 ขั้นตอนที่ 2 — การแยกข้อมูลเชิงโครงสร้าง

ขั้นตอนนี้แปลงผลลัพธ์จากการอ่านเอกสารให้เป็นออบเจกต์ `ReceiptExtraction` สัญญาสาธารณะคือสคีมาและพฤติกรรมของขั้นตอน ส่วนผู้ให้บริการโมเดล ข้อความพรอมป์ นโยบายการกำหนดเส้นทาง งบประมาณโทเคน และเงื่อนไขการลองใหม่ จะถูกจัดการในชั้นปฏิบัติการภายใน

### ขอบเขตการกำหนดเส้นทางโมเดล

Yumo Yumo ดำเนินการแยกข้อมูลเชิงโครงสร้างผ่านอินเทอร์เฟซที่ไม่ขึ้นกับโมเดล นโยบายปฏิบัติการสามารถเลือกเอนจินที่เหมาะสมตามภาษา ความซับซ้อนของเอกสาร สถานะสุขภาพ และสัญญาณด้านคุณภาพ ทั้งค่าน้ำหนัก ลำดับการเลือก และพฤติกรรมสำรองของนโยบายนี้ยังคงเป็นข้อมูลภายใน

### ผลลัพธ์เชิงโครงสร้าง

```json
// ReceiptExtraction
{
  "merchant": {
    "name_raw": "MIGROS T.A.S.",
    "tax_id_raw": "1234567890",
    "address_raw": "...",
    "phone_raw": null
  },
  "captured_at_raw": "17/05/2026 14:23",
  "currency": "TRY",
  "totals": {
    "subtotal": 234.50,
    "tax_total": 42.21,
    "grand_total": 276.71
  },
  "tax_lines": [
    { "rate_pct": 18.0, "base": 200.0, "amount": 36.0 },
    { "rate_pct": 8.0, "base": 77.50, "amount": 6.20 }
  ],
  "payment_method": "credit_card",
  "line_items": [
    {
      "raw_text": "SUT 1L PINAR",
      "qty": 2,
      "unit_price": 23.50,
      "line_total": 47.00,
      "tax_rate_pct": 8.0
    }
  ],
  "quality_band": "medium",
  "extraction_notes": "tax_total reconstructed from tax_lines"
}
```

สคีมาช่วยจัดผลลัพธ์ของโมเดลให้อยู่ในรูปแบบที่ชั้นกฎสามารถตรวจสอบได้ ยอดรวม วันที่ สกุลเงิน รายการสินค้า และฟิลด์ภาษีจะถูกตรวจสอบอีกครั้งในขั้นตอนถัดไป

### การจัดการความสอดคล้อง

หากผลการแยกข้อมูลอยู่ในช่วงคุณภาพต่ำ หรือชั้นกฎพบความไม่สอดคล้อง ไปป์ไลน์สามารถส่งผลลัพธ์ไปให้ตรวจสอบหรือประมวลผลใหม่ได้ การเลือกเส้นทางจะถูกควบคุมผ่านพารามิเตอร์ด้านปฏิบัติการ
