# อรรถประโยชน์ของ INT

## 4.27 พื้นผิวอรรถประโยชน์

INT มีหน้าที่หกประการภายในโปรโตคอล Yumo Yumo สองประการใช้งานอยู่แล้ว อีกสี่ประการวางแผนไว้และจะเปิดใช้งานเมื่อพื้นผิวโปรโตคอลเติบโตเพียงพอ

| หน้าที่ | สถานะ | คำอธิบาย |
|---|---|---|
| **สินทรัพย์รางวัล** | ใช้งานอยู่ | ผู้ใช้ได้รับ INT ผ่านวงจรชีวิตการแปลง bINT → INT (4.4) สำหรับการมีส่วนร่วม Proof of Expense ที่ยืนยันแล้ว |
| **สินทรัพย์ staking** | ใช้งานเมื่อ TGE+1 สัปดาห์ | ผู้ถือครองล็อก INT เข้าพูล staking แบบถ่วงน้ำหนักตามระดับ (4.6) เพื่อรับรางวัลจากการจัดสรร staking incentive |
| **เป้าหมายซื้อคืนและเผา** | ใช้งานเมื่อเริ่มมีรายได้ | คลังสำรองใช้รายได้จากผลิตภัณฑ์ข้อมูลซื้อ INT ในตลาดเปิดและเผา (4.9) สร้างแรงกดดันเงินฝืด |
| **การเผาเพื่อรายงานข้อมูล** | วางแผนไว้ | ธุรกิจที่เข้าถึงรายงานข้อมูลชุมชนแบบรวมต้องเผา INT ตามจำนวนที่กำหนดต่อรายงาน (4.30) |
| **สัญญาณธรรมาภิบาล** | วางแผนไว้ | การส่งสัญญาณถ่วงน้ำหนักด้วย INT สำหรับการตัดสินใจเกี่ยวกับลำดับความสำคัญของผลิตภัณฑ์ข้อมูล การจัดสรรคลังสำรอง และทุนสนับสนุนระบบนิเวศ |
| **การเข้าถึง API แบบวางหลักประกัน** | วางแผนไว้ | ผู้บริโภค API ของผลิตภัณฑ์ข้อมูลที่ไม่ระบุตัวตนอาจต้องวางหลักประกัน INT เข้ากับคีย์การเข้าถึง |

## 4.28 ผลตอบแทนจริง (Real yield)

INT ถูกออกแบบให้เป็นสินทรัพย์ที่ให้ผลตอบแทนจริง มูลค่าระยะยาวมาจากรายได้ภายนอกที่เกิดจากธุรกิจผลิตภัณฑ์ข้อมูล ไม่ใช่จากการปล่อยโทเค็น

กลไกสองประการเชื่อมรายได้ของแพลตฟอร์มกับผู้ถือโทเค็น:

1. **ผลตอบแทนผู้ stake** ส่วนหนึ่งของรายได้สุทธิจากผลิตภัณฑ์ข้อมูลไหลเข้าพูลผลตอบแทนผู้ stake ซึ่งผู้ stake INT สามารถเคลมได้ตามสัดส่วนของยอดวางเดิมพันและน้ำหนักระดับ
2. **ซื้อคืนและเผา** ส่วนหนึ่งของรายได้สุทธิถูกใช้โดยคลังสำรองเพื่อซื้อ INT ในตลาดเปิดและเผาอย่างถาวร

สัดส่วนระหว่างผลตอบแทนผู้ stake และการซื้อคืนและเผาเป็นพารามิเตอร์นโยบายคลังสำรอง ถูกกำกับภายใต้การควบคุมที่อธิบายไว้ใน 4.10 กระแสทั้งสองผูกกับรายได้ภายนอก ไม่ใช่การปล่อยโทเค็น การแยกส่วนนี้หมายความว่าการปล่อย (4.19) ขยายอุปทานอย่างคาดเดาได้ ขณะที่ผลตอบแทนจริงบีบอัดหรือกระจายอุปทานตามผลประกอบการธุรกิจจริง

## 4.29 แหล่งรายได้

รายได้ที่ป้อนเข้ากลไกผลตอบแทนจริงมาจาก:

- **การขายข้อมูลที่ไม่ระบุตัวตน** ข้อมูลระดับใบเสร็จที่ผ่านการ k-anonymize และรวบรวมแล้ว ขายให้แบรนด์ FMCG ผู้ค้าปลีก บริษัทวิจัย และนักพัฒนาผ่านการเข้าถึง API แบบแบ่งระดับ
- **รายได้จากพันธมิตรและการแนะนำ** การเปรียบเทียบราคาที่คลิกผ่านไปยังผู้ค้าปลีกหรือพันธมิตรคูปอง (วางแผนไว้)
- **สมาชิกระดับพรีเมียม** ฟีเจอร์การวิเคราะห์ส่วนตัวขั้นสูงและระบบอัตโนมัติเป้าหมาย (วางแผนไว้)

รายละเอียดการสร้างรายได้และสถาปัตยกรรมการทำให้เป็นนิรนามอธิบายไว้ใน 05 Data Schema and API

## 4.30 การเผาเพื่อรายงานข้อมูล

ธุรกิจที่ซื้อรายงานข้อมูลชุมชนแบบรวมต้องเผา INT ตามจำนวนที่กำหนดสำหรับรายงานแต่ละฉบับที่สร้าง การเผาเกิดขึ้นบนบล็อกเชนและเป็นการถาวร

กลไกนี้ตอบสนองวัตถุประสงค์สองประการ:

1. **แรงกดดันเงินฝืด** รายงานข้อมูลแต่ละฉบับที่ถูกบริโภคจะลบ INT ออกจากอุปทานหมุนเวียนอย่างถาวร สร้างภาวะขาดแคลนจากฝั่งอุปสงค์ตามสัดส่วนการนำผลิตภัณฑ์ข้อมูลไปใช้ในเชิงพาณิชย์
2. **การจัดแนวมูลค่า** ข้อกำหนดการเผาเชื่อมอรรถประโยชน์ของผลิตภัณฑ์ข้อมูลเข้ากับโทเค็นโดยตรง เมื่อธุรกิจจำนวนมากขึ้นบริโภคข้อมูลของ Yumo Yumo INT จำนวนมากขึ้นจะถูกลบออกจากการหมุนเวียน เสริมความสัมพันธ์ระหว่างการใช้งานแพลตฟอร์มกับมูลค่าโทเค็น

จำนวนการเผาต่อรายงานถูกกำหนดโดยนโยบายคลังสำรองและแตกต่างกันตามระดับรายงาน (เช่น ข้อมูลรวมระดับหมวดหมู่ เทียบกับ panel ระดับตะกร้าสินค้าเต็ม) โครงสร้างราคารับประกันว่าต้นทุนการเผายังคงเป็นสัดส่วนเล็กน้อยของมูลค่าเชิงพาณิชย์ของผลิตภัณฑ์ข้อมูล ขณะที่สร้างผลกระทบสะสมต่ออุปทานอย่างมีนัยสำคัญ
