# อัตราส่วนการแปลงและอุปทานหมุนเวียน

## 4.24 อัตราส่วนการแปลง bINT → INT

bINT ชำระเป็น INT ตามอัตราส่วนคงที่ **1:1** การมีส่วนร่วมแต่ละหน่วยมีมูลค่าการแปลงเท่ากันตลอดขอบเขตการปล่อย โดยไม่ขึ้นกับว่ามันถูกได้รับเมื่อใด

อัตราส่วนคงที่ทำให้มูลค่าการแปลงคาดการณ์ได้และขจัดความได้เปรียบด้านเวลาระหว่างการมีส่วนร่วมในช่วงต้นและช่วงหลัง เนื่องจาก bINT แต่ละหน่วยดึง INT มาหนึ่งหน่วยพอดี ราง User Rewards (64.35 พันล้าน INT, 4.17) จึงดูดซับการมีส่วนร่วมได้มากกว่าก่อนถึงเพดาน เมื่อเทียบกับอัตราส่วนที่สูงกว่าในช่วงต้น

การชำระเป็นแบบนอกบล็อกเชน (4.4): เครื่องยนต์แปลง bINT ที่มีสิทธิ์ในแต่ละรอบ (epoch) และผู้ใช้เคลม INT ที่ได้จากตัวกระจาย (distributor) ที่ผ่านการตรวจสอบ เมื่อรางวัลที่มีสิทธิ์รวมในรอบหนึ่งเกินเพดานการปล่อยระดับสากล ผู้เข้าร่วมทุกคนจะถูกปรับลดด้วยตัวคูณ pro-rata เดียวกัน ดังนั้นอัตรารางวัลจะอ่อนลงอย่างเท่าเทียมสำหรับทุกคนแทนที่จะตัดผู้มีส่วนร่วมรายสุดท้ายออก ทั้งค่าเพดานและการคำนวณการปรับลดถูกปรับเทียบในชั้นดำเนินงานและไม่เผยแพร่

## 4.25 ช่วงถือครองและการควบคุมการชำระ

bINT จะเข้าสู่ช่วงถือครองขั้นต่ำก่อนมีสิทธิ์ชำระ หน้าต่างการถือครองให้เวลาชั้นความน่าเชื่อถือ (03) ตรวจจับและตอบสนองต่อรูปแบบที่ผิดปกติก่อนที่ INT จะถูกแจกจ่าย

เพดานสะสมจำกัดจำนวน INT ทั้งหมดที่ชั้นการมีส่วนร่วมสามารถแจกจ่ายได้ (ราง User Rewards, 4.17) ผู้ตรวจสอบอิสระ (4.17) บังคับใช้ค่าคงที่ (invariant) นี้ในแต่ละรอบ พารามิเตอร์เหล่านี้ถูกจัดการในชั้นดำเนินงานและปรับเทียบเพื่อสร้างสมดุลระหว่างประสบการณ์ผู้ใช้กับความปลอดภัยของโปรโตคอล

## 4.26 โมเดลอุปทานหมุนเวียน

INT หมุนเวียนเติบโตจากกระแสขาเข้าหลักสามประการ: การชำระ User Rewards, การปลดล็อก Liquidity และการกระจาย Airdrop เป็นระยะ (4.18) โดยลดลงผ่านการซื้อคืนและเผา (4.9) และการเผาจากการเข้าถึงข้อมูลขององค์กร

ตารางด้านล่างคาดการณ์อุปทานหมุนเวียนภายใต้สามสถานการณ์การเติบโตของ MAU การคาดการณ์เหล่านี้เป็นการสร้างแบบจำลอง ไม่ใช่ข้อผูกมัด

| ปี | สถานการณ์ MAU ต่ำ | สถานการณ์ MAU ฐาน | สถานการณ์ MAU สูง |
|---:|---:|---:|---:|
| TGE | 1,000,000,000 | 1,000,000,000 | 1,000,000,000 |
| 1 | 3,500,000,000 | 5,200,000,000 | 7,400,000,000 |
| 2 | 5,100,000,000 | 8,800,000,000 | 14,000,000,000 |
| 3 | 7,000,000,000 | 13,200,000,000 | 21,500,000,000 |
| 5 | 11,500,000,000 | 22,500,000,000 | 36,000,000,000 |
| 10 | 24,000,000,000 | 42,000,000,000 | 58,000,000,000 |
| 15 | 38,000,000,000 | 60,000,000,000 | 72,000,000,000 |

### สมมติฐาน

- **สัดส่วนลอยตัวเมื่อ TGE** คือสภาพคล่องเริ่มต้น (1,000,000,000) ตรงกับประมาณการใน 4.21 การกระจาย Airdrop เข้าสู่การหมุนเวียนในภายหลังในฐานะกิจกรรมตามการมีส่วนร่วมเป็นระยะ (4.18) ไม่ใช่ที่ TGE
- **MAU ต่ำ:** MAU อยู่ในช่วง 0–10K ในสองปีแรก ถึง 100K ภายในปีที่ 5
- **MAU ฐาน:** MAU ถึง 100K ในปีที่ 1, 1M ภายในปีที่ 3, 5M ภายในปีที่ 5
- **MAU สูง:** MAU ถึง 1M ในปีที่ 1 และรักษาระดับ 5M+ ตั้งแต่ปีที่ 3
- ทุกสถานการณ์สมมติว่ากลไกซื้อคืนและเผาเปิดใช้งานตั้งแต่ปีที่ 2 เป็นต้นไป โดยลดสัดส่วนของอุปทานหมุนเวียนเป็นรายปี อัตราการเผาเป็นฟังก์ชันของรายได้ผลิตภัณฑ์ข้อมูลและนโยบายคลังสำรอง
- Staking ไม่เปิดใช้งานเมื่อเปิดตัว (4.6) โมเดลไม่นับการล็อก staking เป็นแหล่งดูดซับการหมุนเวียนในช่วง v1

การคาดการณ์เหล่านี้แสดงความสัมพันธ์ระหว่างความเร็วของการยอมรับและการขยายตัวของอุปทาน อุปทานหมุนเวียนจริงขึ้นอยู่กับพฤติกรรมการชำระ การดำเนินการเผา และรูปแบบการเติบโตของผู้ใช้ ซึ่งไม่สามารถคาดการณ์ได้อย่างแน่นอน
