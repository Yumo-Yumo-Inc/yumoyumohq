# ข้อมูลสรุปที่ไม่ระบุตัวตนและผลิตภัณฑ์ข้อมูล B2B

## 5.8 ข้อมูลสรุปที่ไม่ระบุตัวตนและผลิตภัณฑ์ข้อมูล B2B

นี่คือพื้นผิวที่สร้างรายได้ B2B และดำเนินการภายใต้การรับประกันความเป็นส่วนตัวที่เคร่งครัด

### กฎการแปลง

การแปลงจาก `receipts + line_items + price_observations` สู่ข้อมูลสรุปที่ไม่ระบุตัวตนเป็นการแปลง **ทิศทางเดียว ทำลาย**:

1. **แทนที่ user_id และ wallet_address** ใช้แฮชที่มีขอบเขตเซสชันแยกจากตัวระบุผู้ใช้
2. **หยาบภูมิศาสตร์** ระดับละเอียดระดับเมืองเท่านั้น พิกัดถูกลบหากเคยบันทึกไว้
3. **หยาบเวลา** ถังรายวันสำหรับดัชนีความถี่สูง ถังรายสัปดาห์สำหรับรายงานระดับหมวดหมู่ รายเดือนสำหรับกลุ่มประชากร
4. **บังคับใช้ k-anonymity** ทุกบันทึกที่เผยแพร่ต้องอยู่ในเซลล์ที่มีผู้มีส่วนร่วมที่แตกต่างกันอย่างน้อยตามจำนวนขั้นต่ำที่กำหนดในสัญญา บนฟิลด์ quasi-identifier (`city`, `merchant_class`, `time_bucket`, `category_path`) เซลล์ที่ต่ำกว่าเกณฑ์ถูกระงับหรือหยาบจนกว่าจะผ่าน
5. **เพิ่มสัญญาณรบกวนความเป็นส่วนตัวแบบต่างที่ปรับเทียบแล้ว** ต่อผลลัพธ์ที่อิงตามจำนวน
6. **สร้างขอบเขตสรุปจาก PoE ที่ได้รับการยืนยัน** ใบเสร็จหน้าคำสั่งซื้อและบันทึกที่รอการตรวจสอบอื่น ๆ ยังคงอยู่ในชั้นความจำ

### ผลิตภัณฑ์ที่มีจำหน่าย

| ผลิตภัณฑ์ | ระดับละเอียด | รีเฟรช | ราคาเชิงบ่งชี้ |
|---|---|---|---|
| **TR Inflation Pulse** | หมวดหมู่ × ภูมิภาค × สัปดาห์ | รายวัน | $X / เดือน (สมัครสมาชิก) |
| **Basket Panel** | สินค้ามาตรฐาน × เมือง × สัปดาห์ | รายวัน | $Y / เดือน |
| **Merchant Benchmarks** | เครือข่าย × หมวดหมู่ × เดือน | รายสัปดาห์ | $Z / เดือน |
| **Custom Cohort Query** | API ต่อการสืบค้น บังคับใช้เกณฑ์ k-anonymity | ตามความต้องการ | $Q / การสืบค้น |

ราคาและระดับละเอียดที่แน่นอนเป็นตัวยึดตำแหน่ง แคตตาล็อกการผลิตจะสรุปก่อนการเปิดตัวเชิงพาณิชย์

### ตัวอย่างการตอบสนอง B2B

```json
// GET /b2b/v1/inflation-pulse?region=istanbul&category=food.dairy&from=2026-05-01&to=2026-05-17
{
  "region": "istanbul",
  "category": "food.dairy",
  "series": [
    { "week_start": "2026-05-04", "index": 100.0, "n_observations": "<n>", "n_distinct_contributors": "<n>" },
    { "week_start": "2026-05-11", "index": 101.7, "n_observations": "<n>", "n_distinct_contributors": "<n>" }
  ],
  "k_anonymity_floor_met": true,
  "methodology_version": "1.0.0"
}
```

ทุกการตอบสนอง B2B มี `n_distinct_contributors` เพื่อให้ผู้ซื้อสามารถตรวจสอบว่าพื้น k-anonymity ถูกบรรลุ เซลล์ที่จะต่ำกว่าพื้นจะส่งคืนเป็น `suppressed: true` โดยไม่มีค่า ค่าเกณฑ์เฉพาะจัดการอยู่ในชั้นปฏิบัติการภายใน

### ฟิลด์นอกขอบเขต

- ฟิลด์ใดก็ตามที่ผูกกับผู้ใช้รายเดียว
- บันทึกใดก็ตามที่ต่ำกว่าเกณฑ์ k-anonymity สำหรับเซลล์ที่สืบค้น
- พิกัด ที่อยู่ หมายเลขโทรศัพท์
- ข้อมูลเมตาเครื่องมือการชำระเงิน
- IDs ที่ไม่ระบุตัวตนแต่เชื่อมโยงได้ข้ามการสืบค้น (ทุกการสืบค้นสร้างแฮชเซสชันใหม่)

### เปรียบเทียบกับแผงข้อมูลที่มีอยู่

ผลิตภัณฑ์ข้อมูล B2B ของ Yumo Yumo แข่งขันกับ Nielsen, GfK, Kantar และ SimilarWeb สำหรับผู้ซื้อรายเดียวกัน เมื่อเทียบกับแผงข้อมูลเหล่านั้น:

- **ระดับใบเสร็จ** ไม่ใช่การสำรวจ — ข้อผิดพลาดการวัดน้อยกว่า
- **ความถี่รีเฟรชสูงกว่า** — รายวัน เทียบกับ รายสัปดาห์/เดือน
- **การครอบคลุมตลาดเกิดใหม่** — TR เป็นอันดับแรก การครอบคลุม TR บางที่สุดในบรรดาผู้ดำรงตำแหน่ง
- **ต้นทุนต่อบันทึกต่ำกว่า** — แผงข้อมูลสร้างจากกิจกรรมผู้ใช้ที่มีอยู่ ไม่ใช่ผู้เข้าร่วมที่ได้รับค่าจ้าง

การแลกเปลี่ยน: แผงข้อมูลของ Yumo Yumo มีขนาดเล็กกว่าเมื่อเปิดตัว และเอียงไปทางประชากรผู้ใช้งานล่วงหน้า
