# การสร้างดัชนีและการแบ่งพาร์ติชัน

## 5.12 การสร้างดัชนีและการแบ่งพาร์ติชัน

`receipts` และ `receipt_line_items` ถูกแบ่งพาร์ติชันตามเดือน `uploaded_at` ช่วงผลิตภัณฑ์ที่ใช้งานอยู่อยู่ในชั้นข้อมูลร้อน พาร์ติชันที่เก่ากว่าเลื่อนไปยังชั้นการวิเคราะห์ที่มีต้นทุนต่ำกว่า คลาสดัชนีที่ใช้งาน:

| ตาราง | ดัชนี | การใช้งาน |
|---|---|---|
| `receipts` | `(user_id, uploaded_at DESC)` | แสดงรายการใบเสร็จของผู้ใช้ |
| `receipts` | `(merchant_id, uploaded_at DESC)` | คิวผู้ค้า |
| `receipt_line_items` | `(canonical_product_id, uploaded_at DESC)` | การสังเกตราคา |
| `price_observations` | `(canonical_product_id, observed_at)` | ชีพจรเงินเฟ้อ |
| `canonical_products` | `embedding_vector` (approximate nearest-neighbour) | การจับคู่ขั้นตอน 4 |
| `bint_ledger` | `(user_id, created_at DESC)` | การสืบค้นยอดคงเหลือ |

ดัชนีเวกเตอร์เป็นดัชนีที่แพงที่สุดในการสร้างใหม่ และเป็นปัจจัยจำกัดในการเติบโตของแคตตาล็อกมาตรฐาน — 02 2.7 ระบุสิ่งนี้เป็นคันโยกต้นทุน เครื่องมือสร้างดัชนีเฉพาะและพารามิเตอร์การปรับแต่งจัดการอยู่ในชั้นปฏิบัติการภายใน
