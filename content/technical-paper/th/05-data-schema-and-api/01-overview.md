# 05 — รูปแบบข้อมูลและ API

ส่วนนี้กำหนดรูปแบบข้อมูลสาธารณะและพื้นผิว API ของ Yumo Yumo ผลลัพธ์ของไพพ์ไลน์ใน 02 การตัดสินใจด้านความน่าเชื่อถือใน 03 และการบัญชีรางวัลใน 04 จะถูกจับคู่กับรูปร่างเอนทิตีและเหตุการณ์ที่กำหนดไว้ที่นี่

รูปแบบข้อมูลถูกเผยแพร่เพื่อให้ฝ่ายต่าง ๆ ในโปรโตคอลสามารถอ่านบันทึกเดียวกันในแบบเดียวกัน กลยุทธ์การสร้างดัชนีทางกายภาพ การเลือกผู้ให้บริการ การเคลื่อนย้ายข้อมูลร้อน/เย็น และพารามิเตอร์เชิงพาณิชย์ของผลิตภัณฑ์ B2B ยังคงอยู่ในเอกสารปฏิบัติการ

## 5.0 หลักการรูปแบบข้อมูล

| หลักการ | ผลที่ตามมา |
|---|---|
| บันทึกที่มีประเภท | ทุกออบเจกต์สาธารณะมีเวอร์ชันและผูกกับสคีมา |
| บัญชีแยกประเภตแบบเหตุการณ์ | การบัญชีรางวัลสืบเนื่องมาจากเหตุการณ์ประวัติศาสตร์ที่สามารถตรวจสอบได้ |
| การแยกข้อมูล | ข้อมูลส่วนบุคคล ข้อมูลสรุป และสรุปบนเชนอยู่ในชั้นต่างหาก |
| API ที่มีเวอร์ชัน | การเพิ่ม/ลบฟิลด์ปฏิบัติตามกฎความเข้ากันได้ย้อนหลัง |
