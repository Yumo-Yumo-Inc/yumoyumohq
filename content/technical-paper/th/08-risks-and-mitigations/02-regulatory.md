# ความเสี่ยงด้านกำกับดูแล

## 8.2 พื้นผิว

Yumo Yumo ดำเนินงานข้ามพื้นผิวผลิตภัณฑ์ข้อมูล การมีส่วนร่วมของผู้ใช้ และเศรษฐกิจโทเค็น ดังนั้นหลายกรอบกำกับดูแลจึงสัมผัสโปรโตคอล:

| กรอบ | พื้นผิวทางเทคนิค | หลักการควบคุมสาธารณะ |
|---|---|---|
| KVKK | ข้อมูลประจำตัวผู้ใช้ เนื้อหาใบเสร็จ และสต็อกผู้ประมวลผลในตุรกี | การย่อข้อมูลให้เหลือน้อยที่สุดและบันทึกกระบวนการทางกฎหมาย |
| GDPR | ข้อมูลผู้ใช้ EU / EEA และผลิตภัณฑ์ข้อมูลสรุป | กระบวนการสิทธิผู้ใช้และวินัยการเผยแพร่ข้อมูลสรุป |
| MiCA | การจัดหมวดหมู่ INT เป็นสินทรัพย์คริปโตใน EU | การจดทะเบียนภูมิภาคและที่ปรึกษาทางกฎหมาย |
| US token classification | การออกแบบและแบบจำลองการจัดจำหน่ายโทเค็นอรรถประโยชน์ INT | การปล่อยที่เชื่อมโยงกับการมีส่วนร่วม การถือสิทธิสาธารณะ และท่าทีอรรถประโยชน์ |
| Tax classification | รางวัลผู้ใช้ รายได้องค์กร VAT / ภาษีการขาย | การบัญชีและกระบวนการรายงานเฉพาะภูมิภาค |

กรอบเหล่านี้สามารถกำหนดหน้าที่การปฏิบัติตามที่แตกต่างกันให้กับกลไกทางเทคนิคเดียวกันในภูมิภาคต่างกัน เอกสารทางเทคนิคกำหนดพื้นผิวสถาปัตยกรรมที่สามารถรองรับหน้าที่เหล่านั้น

## 8.3 แบบจำลองการควบคุม

**การย่อข้อมูลให้เหลือน้อยที่สุด** เนื้อหาใบเสร็จถูกเก็บไว้ในชั้นข้อมูลนอกเชน (04 §4.16) ชั้นบนเชนถ่ายทอดเหตุการณ์ mint bINT การชำระเงิน INT และคำมั่นสัญญาสรุป หลักฐานความสมบูรณ์ถูกสร้างขณะที่ประวัติการใช้จ่ายของผู้ใช้ยังคงอยู่นอกข้อมูลเชนสาธารณะ

**นโยบายการเผยแพร่ข้อมูลสรุป** ผลิตภัณฑ์ข้อมูล B2B ปฏิบัติตามกฎ k-anonymity และการเผยแพร่ข้อมูลสรุป (05 §5.8) ข้อมูลที่เผยแพร่สร้างสัญญาณระดับภูมิภาค / ช่วงเวลา / หมวดหมู่ แทนที่จะเปิดเผยใบเสร็จรายบุคคล

**โครงสร้างองค์กร** Yumo Yumo Inc. เป็นบริษัท Delaware (00 §0.1) การจดทะเบียนภูมิภาค การเป็นตัวแทน และความสัมพันธ์ผู้ให้บริการดำเนินการด้วยที่ปรึกษาทางกฎหมายและแผนการเปิดตัวผลิตภัณฑ์

**ท่าทีการจัดหมวดหมู่โทเค็น** การออกแบบเศรษฐกิจของ INT สร้างขึ้นรอบอรรถประโยชน์และกลไกการมีส่วนร่วม: การปล่อยเชื่อมโยงกับการมีส่วนร่วมที่วัดได้ (4.3) รางวัล staking มาจากพูลสาธารณะ (4.6) และ BBB ได้รับทุนจากรายได้การดำเนินงาน (4.9)

## 8.4 วิวัฒนาการ

เมื่อการแปลภาษาทีละขั้นตอนคืบหน้า ความรับผิดชอบด้านการปฏิบัติตามแนบอยู่กับแผนการย้ายอำนาจ (00 §0.2, 04 §4.10) โครงสร้างองค์กร การดูแลข้อมูล และบทบาทบริการโทเค็นถูกออกแบบให้ย้ายข้ามโครงสร้างภูมิภาคภายใต้สถาปัตยกรรมเดียวกัน
