# รายการองค์ประกอบ

## 1.2 รายการองค์ประกอบ

รายการนี้แสดงความรับผิดชอบขององค์ประกอบโปรโตคอลและสัญญาที่เปิดเผยต่อสาธารณะ

| องค์ประกอบ | Responsibility | สัญญาสาธารณะ |
|---|---|---|
| แอปพลิเคชันของลูกค้า | บันทึกใบเสร็จ ลายเซ็นกระเป๋าเงิน ตัวอย่างสำหรับผู้ใช้ | การลงนามของผู้ใช้เกิดขึ้นที่อุปกรณ์ การอัปโหลดเริ่มต้นจากการกระทำที่ชัดเจนของผู้ใช้ |
| พื้นผิว API | ข้อมูลประจำตัว การประสานงานอัปโหลด การเริ่มต้นท่อ การสอบถามสถานะ | พื้นผิว REST/SDK ที่เสถียร สถานะขั้นตอนและหมวดหมู่ข้อผิดพลาด |
| ท่อประมวลผลใบเสร็จ | แปลงอินพุตภาพ/PDF เป็นบันทึกใบเสร็จที่มีโครงสร้าง | ผลลัพธ์ระหว่างกลางที่มีประเภท สถานะการตรวจสอบ การอ้างอิงสินค้ามาตรฐานและผู้ค้า |
| ชั้นความน่าเชื่อถือ | สร้างคุณสมบัติรางวัลจากใบเสร็จและสัญญาณผู้ใช้ | ช่วงสาธารณะ หมวดหมู่การตัดสินใจ และการปรับเทียบที่จัดการในการดำเนินงานภายใน |
| บัญชีแยกประเภทและการบัญชีรางวัล | เก็บเหตุการณ์ bINT/ePoints เป็นสตรีมการบัญชีที่เปลี่ยนแปลงไม่ได้ | โมเดลเหตุการณ์แบบ append-only บันทึกการชำระเงินที่สามารถตรวจสอบได้ |
| ชั้นผลิตภัณฑ์ข้อมูล | สร้างข้อมูลสรุปที่ไม่ระบุตัวตนจากการสังเกตใบเสร็จ | การแยกข้อมูลส่วนบุคคล เกณฑ์ k ผลลัพธ์สรุปที่มีเวอร์ชัน |
| โปรแกรมบนเชน | สถานะโทเคน การ stake การกำหนดเส้นทางคลัง และคำมั่นสัญญาทางการเข้ารหัสลับ | อินเทอร์เฟซโปรแกรมที่สามารถตรวจสอบได้ และที่อยู่โปรแกรมที่เผยแพร่ |
| ระนาบควบคุมการดำเนินงาน | การติดตาม โควต้า การกำหนดเส้นทางผู้ให้บริการ การตอบสนองเหตุการณ์ | สรุปสถานะสาธารณะ runbook เกณฑ์ และรายละเอียดการกำหนดเส้นทางยังคงเป็นส่วนตัว |

การแยกส่วนนี้มีความสำคัญต่อความปลอดภัย เอกสารสาธารณะทำให้สถาปัตยกรรมสามารถตรวจสอบได้ ขณะที่การรวมกันของโครงสร้างพื้นฐาน เกณฑ์ และขั้นตอนการตอบสนองจัดการในการดำเนินงานภายใน
