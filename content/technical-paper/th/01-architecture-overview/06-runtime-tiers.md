# ระดับรันไทม์

## 1.5 ระดับรันไทม์

Yumo Yumo มีสามระดับรันไทม์ แต่ละระดับมีงบประมาณที่แตกต่างกัน

| ระดับ | สิ่งที่ทำงานที่นี่ | งบประมาณความล่าช้า | งบประมาณต้นทุน | โหมดความล้มเหลว |
|---|---|---|---|---|
| ซิงโครนัส (คำขอ) | OCR → LLM → regex → canonical match → คะแนนความน่าเชื่อถือ | P95 < ~5 วินาที end-to-end | < 0.02 USD / ใบเสร็จ | แสดงตัวอย่างที่ลดคุณภาพ + ลองใหม่ |
| อะซิงโครนัส (การชำระเงิน) | mint bINT การอัปเดตระดับ NFT | < 1 ชั่วโมงจากการตรวจสอบ | ค่าธรรมเนียม Solana | เลื่อนไปชุดถัดไป |
| ชุดรายวัน | การจัดกลุ่ม canonical ใหม่ การคำนวณคะแนนสุขภาพใหม่ การส่งออกที่ไม่ระบุตัวตน คิว BBB | ช่วงกลางคืน | งบประมาณกลุ่มการคำนวณ | ดำเนินการกับ snapshot ของเมื่อวาน |

การแยกระดับเหล่านี้ช่วยให้ประสบการณ์ที่มุ่งสู่ผู้ใช้ยังคงรวดเร็ว แม้ว่าระบบพื้นฐานจะมีต้นทุนสูงหรือช้า ต้นทุนการ mint บน Solana (ข้อจำกัดสำหรับประสบการณ์ผู้ใช้บนเชนแบบ sub-second) ถูกกระจายตามชุด
