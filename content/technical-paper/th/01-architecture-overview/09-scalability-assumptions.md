# สมมติฐานความสามารถในการขยายตัว

## 1.8 สมมติฐานความสามารถในการขยายตัว

ส่วนนี้แสดงความจุผ่านตัวแปร workload สถาปัตยกรรมประเมินจำนวนผู้ใช้ ความหนาแน่นของใบเสร็จ จำนวนรายการ อัตราการลองใหม่ และนโยบายชุดร่วมกัน

| สัญลักษณ์ | ความหมาย |
|---|---|
| `U` | ผู้ใช้งานประจำวัน |
| `r` | ใบเสร็จเฉลี่ยต่อผู้ใช้ต่อวัน |
| `a` | สัดส่วนใบเสร็จที่ส่งมาประมวลผล |
| `L` | รายการเฉลี่ยต่อใบเสร็จ |
| `v` | สัดส่วนใบเสร็จที่ตรวจสอบแล้วและมีสิทธิได้รับรางวัล |
| `e` | บันทึกเหตุการณ์เฉลี่ยต่อใบเสร็จ |
| `ρ_ocr` | อัตราการลองใหม่ของ OCR |
| `ρ_llm` | อัตราการลองใหม่หรืออัตราความสอดคล้องตนเองของ LLM |
| `B` | ขนาดชุดการชำระเงินบนเชน |

ปริมาณใบเสร็จที่ประมวลผลรายวัน:

```text
R_d = U × r × a
```

การเติบโตของข้อมูลร้อนโดยประมาณรายเดือน:

```text
Rows_m ≈ 30 × R_d × (1 + L + e)
```

ปริมาณการเรียกโมเดลรายวัน:

```text
OCR_d ≈ R_d × (1 + ρ_ocr)
LLM_d ≈ R_d × (1 + ρ_llm)
```

ปริมาณรางวัลและการชำระเงินรายวัน:

```text
Verified_d ≈ R_d × v
Onchain_batches_d ≈ ceil(Verified_d / B)
```

รูปแบบต้นทุนผันแปรสาธารณะ:

```text
Cost_d ≈ OCR_d × c_ocr + LLM_d × c_llm + Storage_d × c_storage + Settlement_d × c_chain
```

การตัดสินใจด้านความจุทำโดยการวัด `U` `r` `L` อัตราการลองใหม่ และนโยบายชุดร่วมกัน เกณฑ์ตัวเลขและสัมประสิทธิ์ต้นทุนเฉพาะผู้ให้บริการยังคงอยู่ในการวางแผนปฏิบัติการ
