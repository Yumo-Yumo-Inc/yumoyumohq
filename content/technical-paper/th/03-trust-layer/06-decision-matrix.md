# เมทริกซ์การตัดสินใจ

## 3.11 จากคะแนนสู่ผลลัพธ์

เมื่อใบเสร็จได้รับช่วงความน่าเชื่อถือ และผู้ใช้มีสแน็ปช็อตสุขภาพล่าสุดแล้ว ชั้นนี้จะกำหนดให้ใบเสร็จไปสู่หนึ่งในสี่ผลลัพธ์

| ผลลัพธ์ | สิ่งที่ผู้ใช้เห็น | สิ่งที่บัญชีแยกประเภทบันทึก |
|---|---|---|
| **ยอมรับ — เครดิตเต็ม** | ตัวอย่างที่ตรวจสอบแล้ว + จำนวน bINT เต็มสำหรับใบเสร็จนี้และผู้ใช้นี้ | `receipt.status = "verified"` เครดิตเต็ม พร้อมระบุตระกูลสัญญาณ |
| **ยอมรับ — เครดิตลดลง** | ตัวอย่างที่ตรวจสอบแล้ว + จำนวน bINT ที่น้อยลง โดยไม่เพิ่มขั้นตอนรบกวนผู้ใช้ | `receipt.status = "verified"` เครดิตบางส่วน พร้อมหมวดหมู่เหตุผลของการลด |
| **ระงับเพื่อตรวจสอบ** | "เรากำลังตรวจสอบรายการนี้ โดยปกติผลจะออกภายในหนึ่งวัน" | `receipt.status = "under_review"` เข้าคิวในเวิร์กโฟลว์การอุทธรณ์ (3.12) |
| **ปฏิเสธ** | ข้อความที่ชัดเจน เข้าใจง่าย และ 0 bINT | `receipt.status = "rejected"` พร้อมหมวดหมู่เหตุผลในการปฏิเสธ |

ผลลัพธ์ที่สี่สงวนไว้สำหรับกรณีที่อยู่นอกขอบเขตความเป็นไปได้ของใบเสร็จที่สุจริต เช่น ภาพที่ถูกระบุโดยการตรวจสอบความแท้ของสื่อสังเคราะห์ เอกสารที่เขียนด้วยมือ หรือใบเสร็จซ้ำที่เคยให้เครดิตแก่ผู้ใช้อื่นไปแล้วพร้อมหลักฐานที่ขัดแย้งกัน

## 3.12 คิวอุทธรณ์

ใบเสร็จที่ถูกระงับเพื่อตรวจสอบจะเข้าสู่คิวภายใต้เป้าหมายด้านระยะเวลาการปฏิบัติการ ผู้ตรวจสอบ (ในช่วงแรกคือทีมปฏิบัติการ และต่อมาคือพูลชุมชนที่ได้รับ Proof of Contribution) จะเห็น:

- ภาพใบเสร็จและระเบียนที่แยกข้อมูลแล้ว
- ช่วงความน่าเชื่อถือและรายการตระกูลสัญญาณที่มีส่วนต่อการตัดสินใจ
- ภาพรวมอย่างย่อของประวัติล่าสุดของผู้ใช้
- การดำเนินการ 3 แบบ: **อนุมัติเต็มจำนวน**, **อนุมัติแบบลดจำนวน**, **คงคำปฏิเสธเดิม**

ผู้ตรวจสอบจะเห็นตระกูลสัญญาณเดียวกับที่บล็อกของใบเสร็จบันทึกไว้ สิ่งนี้ช่วยให้การตัดสินใจของผู้ตรวจสอบสอดคล้องกับการออกแบบของชั้น และสนับสนุนให้เกิดการตัดสินใจที่สม่ำเสมอ

หากผู้ตรวจสอบกลับคำแนะนำของชั้น การกลับคำนั้นจะถูกบันทึกและนำไปใช้ในรอบการปรับเทียบครั้งถัดไป

## 3.13 สิ่งที่ผู้ใช้สามารถทำได้

ผู้ใช้ที่ใบเสร็จถูกปฏิเสธจะเห็นคำอธิบายในระดับหมวดหมู่ และในกรณีที่เหมาะสมจะมีแนวทางให้ดำเนินการด้วยตนเอง เช่น ถ่ายใบเสร็จใหม่ในแสงที่ดีกว่า ติดต่อฝ่ายสนับสนุนพร้อมหลักฐานการชำระเงิน หรือยอมรับผลการปฏิเสธ เหตุผลในระดับสัญญาณจะยังคงอยู่ในการกำหนดค่าความน่าเชื่อถือภายใน

ผู้ใช้ที่คะแนนสุขภาพลดลงสามารถฟื้นตัวได้ด้วยการส่งใบเสร็จที่สะอาดอย่างต่อเนื่อง การฟื้นตัวถูกออกแบบให้ค่อยเป็นค่อยไปโดยตั้งใจ ระบบให้รางวัลกับพฤติกรรมที่ดีอย่างสม่ำเสมอ มากกว่าการเร่งทำแบบฉับพลัน
