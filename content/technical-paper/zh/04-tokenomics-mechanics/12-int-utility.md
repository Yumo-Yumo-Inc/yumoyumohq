# INT 效用

## 4.27 效用表面

INT 在 Yumo Yumo 协议内承载六项功能。其中两项已上线；四项为计划中，随协议表面成熟后启用。

| 功能 | 状态 | 描述 |
|---|---|---|
| **奖励资产** | 已上线 | 用户通过 bINT → INT 转换生命周期（4.4）为已验证的费用证明贡献获得 INT |
| **质押资产** | TGE+1 周上线 | 持有者将 INT 锁入等级加权质押池（4.6）以从质押激励分配中获得奖励 |
| **回购销毁标的** | 收入启动时上线 | 金库使用数据产品收入在公开市场购买 INT 并销毁（4.9），产生通缩压力 |
| **数据报告销毁** | 计划中 | 访问聚合社区数据报告的企业须按报告销毁指定数量的 INT（4.30） |
| **治理信号** | 计划中 | INT 加权信号，用于数据产品优先级、金库分配和生态系统拨款的决策 |
| **绑定 API 访问** | 计划中 | 匿名化数据产品的 API 消费者可能需要将 INT 绑定至其访问密钥 |

## 4.28 实际收益

INT 被设计为实际收益资产。其长期价值捕获来自数据产品业务产生的外部收入，而非代币发行。

两个机制将平台收入与代币持有者连接：

1. **质押者收益。** 净数据产品收入的一部分流入质押者收益池，由 INT 质押者按其质押量和等级权重按比例领取。
2. **回购销毁。** 净收入的一部分由金库用于在公开市场购买 INT 并永久销毁。

质押者收益与回购销毁之间的分配是金库政策参数，在 4.10 描述的控制下治理。两个流都与外部收入挂钩，而非代币发行。这种分离意味着发行（4.19）以可预测的方式扩张供应，而实际收益则根据实际业务表现压缩或分配供应。

## 4.29 收入来源

为实际收益机制提供资金的收入来自：

- **匿名化数据销售。** 经 k-anonymization 处理的、聚合的收据级数据，通过分级 API 访问出售给快消品牌、零售商、研究机构和开发者。
- **联盟与推荐收入。** 价格比较的点击跳转至零售商或优惠券合作伙伴（计划中）。
- **高级订阅。** 高级个人分析和目标自动化功能（计划中）。

收入生成细节和匿名化架构描述于 05 数据架构与 API。

## 4.30 数据报告销毁

购买聚合社区数据报告的企业须为其生成的每份报告销毁指定数量的 INT。销毁在链上进行且不可逆。

该机制服务于两个目的：

1. **通缩压力。** 每份消费的数据报告永久地从流通供应中移除 INT，创造与数据产品商业采用成正比的需求侧稀缺性。
2. **价值对齐。** 销毁要求将数据产品的效用直接与代币绑定。随着更多企业消费 Yumo Yumo 数据，更多 INT 被从流通中移除，强化了平台使用与代币价值之间的关系。

每份报告的销毁数量由金库政策设定，并根据报告层级（例如品类级聚合 vs. 完整购物篮级面板）而有所不同。定价结构确保销毁成本仅占数据产品商业价值的一小部分，同时对供应产生有意义的累积影响。
