# 代币类别

## 4.1 四个类别

Yumo Yumo 以四个资产类别运作，每个类别承担独立的角色。其中只有两个是链上代币；另外两个是链下记录。

| 类别 | 形态 | 转移模型 | 角色 |
|---|---|---|---|
| **INT** | Solana 上的 SPL 代币 | 市场可转让 | 协议层协调、质押、生态激励。供应量参数存放于 Vision Paper。 |
| **bINT** | 链下会计单位（营运层） | 通过既定生命周期结算为 INT | 工作与奖励之间的贡献会计层。 |
| **ePoints** | 链下、以美元计价的记录 | 应用内洞察额度 | 每张已验证收据所揭示的家庭层隐性成本记录。 |
| **Foundation NFT（Yumbie）** | Token-2022 不可转让资产 | 不可转让 | 持久身份。在 Vision Paper 定义的里程碑处演化为 Smart Agent。 |

### 为何设四个类别

Vision Paper 解释了用户体验层面的原因。机制层面的原因则是关注点分离：

- INT 在市场与交易所间流通；它可转让且可替代。
- bINT 衡量贡献并结算为 INT；它是链下单位，因此会计可在不进行链上迁移的情况下演进。
- ePoints 以自身的链下记录承载经济洞察信号，因此在 INT 供应量保持固定的同时，用户分析仍可增长。
- Foundation NFT 以不可转让的 Token-2022 资产承载身份连续性，每个钱包一个。

## 4.2 权限结构

权限因类别为链上或链下而不同。

- **INT 铸造权限** — 仅在创世时铸造完整供应量之前持有，之后关闭。创世后无法再铸造任何 INT；分发是通过经审计的分发器（4.15）从金库转账。
- **INT 金库与销毁** — 由 Squads 多签持有，对分发根签名、金库转移与储备回收设有分离的核准。
- **bINT 与 ePoints** — 营运层中的链下会计单位。它们没有链上铸造或冻结权限；其余额通过 4.4 中的生命周期结算为 INT。
- **Foundation NFT** — 带不可转让扩展的 Token-2022，由后端为每个钱包一次性铸造。不可转让性在代币程序层强制执行。

将 bINT 与 ePoints 保留在链下，从贡献路径中移除了逐事件的链上权限；创世后唯一持续存在的 INT 层权限是多签对金库、分发根与销毁的控制。
