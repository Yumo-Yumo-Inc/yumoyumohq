# 安全邊界

## 1.7 安全邊界

Yumo Yumo 的安全模型將各層所持有的資料與權限分離。公開文件以可審計的方式說明權限與資料邊界。

| 邊界 | 持有內容 | 權限分離 |
|---|---|---|
| 使用者裝置 | 錢包簽名、選取的收據檔案、本地預處理 | 使用者簽名保留於裝置端 |
| 應用服務 | 工作階段、上傳編排、管線作業、狀態事件 | 應用服務承載工作階段與管線權限 |
| 資料平面 | 化名收據記錄、衍生觀測資料、獎勵分類帳 | 資料平面承載記錄與分類帳完整性 |
| 鏈上層 | 代幣狀態、質押/金庫權限、密碼學承諾 | 鏈上層承載代幣與承諾狀態 |
| 營運控制平面 | 監控、配額、事件回應 | 營運控制平面管理防禦參數 |

在此模型中，使用者資料、獎勵會計與鏈上權限各自居於獨立層級。跨邊界轉換透過型別化事件與可審計記錄進行；簽署流程、緊急應變手冊與門檻值則由私有營運文件管理。
