# 可擴展性假設

## 1.8 可擴展性假設

本節以工作負載變數表達容量。架構將使用者數量、收據密度、品項數量、重試率與批次策略一併評估。

| 符號 | 意義 |
|---|---|
| `U` | 每日活躍使用者 |
| `r` | 每位使用者每日平均收據數 |
| `a` | 提交處理的收據佔比 |
| `L` | 每張收據平均品項數 |
| `v` | 符合獎勵資格的已驗證收據佔比 |
| `e` | 每張收據平均事件記錄數 |
| `ρ_ocr` | OCR 重試率 |
| `ρ_llm` | LLM 重試或自我一致性率 |
| `B` | 鏈上結算批次大小 |

每日處理收據量：

```text
R_d = U × r × a
```

每月熱資料成長約略值：

```text
Rows_m ≈ 30 × R_d × (1 + L + e)
```

每日模型呼叫量：

```text
OCR_d ≈ R_d × (1 + ρ_ocr)
LLM_d ≈ R_d × (1 + ρ_llm)
```

每日獎勵與結算量：

```text
Verified_d ≈ R_d × v
Onchain_batches_d ≈ ceil(Verified_d / B)
```

公開變動成本公式：

```text
Cost_d ≈ OCR_d × c_ocr + LLM_d × c_llm + Storage_d × c_storage + Settlement_d × c_chain
```

容量決策透過共同測量 `U`、`r`、`L`、重試率與批次策略制定。數值門檻與供應商特定成本係數保留於營運規劃中。
