# 執行階層

## 1.5 執行階層

Yumo Yumo 具有三個執行階層，各有不同的資源預算。

| 階層 | 執行內容 | 延遲預算 | 成本預算 | 故障模式 |
|---|---|---|---|---|
| 同步（請求層）| OCR → LLM → 規則 → 標準配對 → 信任評分 | P95 < 約 5 秒端到端 | < 0.02 美元 / 收據 | 顯示降級預覽 + 重試 |
| 非同步（結算層）| bINT 鑄造、NFT 等級更新 | 驗證後 < 1 小時 | Solana 手續費 | 延至下一批次 |
| 每日批次 | 標準重新分群、健康分數重算、匿名化匯出、BBB 佇列 | 夜間視窗 | 運算池預算 | 以昨日快照運作 |

將這些階層分離，可讓面向使用者的體驗保持快速，即使底層系統昂貴或緩慢。Solana 鑄造成本（亞秒級鏈上使用者體驗的瓶頸）透過批次攤銷。
