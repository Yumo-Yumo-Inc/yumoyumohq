# 元件清單

## 1.2 元件清單

本清單列出協議元件的職責及其公開的契約介面。

| 元件 | 職責 | 公開契約 |
|---|---|---|
| 客戶端應用程式 | 收據拍攝、錢包簽名、使用者預覽 | 使用者簽名於裝置端完成；上傳由明確的使用者操作觸發 |
| API 介面 | 身分驗證、上傳編排、管線啟動、狀態查詢 | 穩定的 REST/SDK 介面；階段狀態與錯誤類別 |
| 收據處理管線 | 將圖片/PDF 輸入轉換為結構化收據記錄 | 型別化的中間輸出、驗證狀態、標準商品與商家引用 |
| 信任層 | 根據收據與使用者訊號產生獎勵資格 | 公開的級距（bands）、決策類別與校準由內部營運管理 |
| 分類帳與獎勵會計 | 將 bINT/ePoints 事件儲存為不可變的會計流 | 僅追加（append-only）事件模型、可審計的結算記錄 |
| 資料產品層 | 從收據觀測資料產出匿名化彙總 | 個人資料分離、k-閾值、版本化彙總輸出 |
| 鏈上程式 | 代幣狀態、質押、金庫路由與密碼學承諾 | 可審計的程式介面與已公布的程式地址 |
| 營運控制平面 | 監控、配額、供應商路由、事件回應 | 公開狀態摘要；運行手冊、門檻與路由細節保持私有 |

這種分離對安全性至關重要：公開文件使架構可審計，而基礎設施組合、門檻與回應步驟則由內部營運管理。
