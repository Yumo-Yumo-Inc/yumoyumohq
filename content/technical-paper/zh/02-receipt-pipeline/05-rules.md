# 階段 3 — 規則

## 2.6 階段 3 — 正規表示式與規則層

正規表示式層**永遠**在 LLM 之後執行。LLM 是快速的模式辨識器；正規表示式則是確定性的驗證器。

### 正規表示式能捕捉、LLM 會遺漏的項目

規則層在四個類別上增加確定性驗證：

1. **總額核對。** LLM 偶爾會四捨五入或標準化總額。規則層從 OCR 文字重新提取列印的總額，並在出現分歧時覆蓋 LLM 的值。
2. **日期標準化。** 收據使用多種區域日期格式與在地化月份名稱。所有變體統一收斂至 ISO 8601。
3. **幣別消歧。** 混合幣別代號依頻率與商家所在地解析。
4. **稅行偵測。** 在地化稅率行（例如土耳其的 KDV）從列印文字偵測，而非由 LLM 推斷。

### 規則目錄

目錄依類別組織：總額、稅額、日期、幣別與商家識別碼。每個類別包含我們營運語言的在地化規則家族。具體模式與每條規則的權重由內部營運層管理。

### 信心度提升

每項驗證通過的規則命中都會提升正規表示式層的信心分數。順利核對的收據獲得較高分數；未解析總額的收據則帶有核對落差，信任評分器將其視為輸入之一。該輸入的權重由內部營運層管理。
