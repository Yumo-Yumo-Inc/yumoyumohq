# 階段 1 — 文件讀取

## 2.4 階段 1 — 文件讀取層

本階段從收據圖片或 PDF 輸入中提取文字區塊、閱讀順序與位置資訊。公開契約為本階段的標準化輸出，而非供應商名稱。

### 輸出標準化

文件讀取引擎可回傳不同格式。管線將其標準化為單一內部格式：

```json
// DocumentReadResult
{
  "raw_text": "MIGROS\nFIS NO: 4521\n...",
  "blocks": [
    {
      "text": "MIGROS",
      "bbox": { "x": 120, "y": 40, "w": 200, "h": 50 },
      "confidence_band": "high",
      "reading_order": 0
    }
  ],
  "quality_band": "high",
  "detected_languages": ["tr"],
  "page_count": 1
}
```

區塊依閱讀順序排序後傳遞至下一階段作為確定性輸入。因此模型提取不會與任何原始供應商回應格式耦合。

### 品質訊號

文件讀取階段將品質級距與錯誤類別攜帶至後續階段。在低品質情況下，管線可依營運策略重新處理、請求使用者提供新圖片，或以較低信心度繼續處理。

這在保留公開技術契約的同時，避免了容易被逆向工程的門檻與備援細節。
