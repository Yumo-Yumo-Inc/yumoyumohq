# 05 — 資料架構與 API

本節定義 Yumo Yumo 的公開資料綱要與 API 介面。02 的管線輸出、03 的信任決策與 04 的獎勵會計均對應至此處定義的實體與事件形狀。

綱要公開發布，使協議各方能以相同方式讀取相同記錄。實體索引策略、供應商選擇、熱/冷資料遷移以及 B2B 產品的商業參數保留於營運文件中。

## 5.0 綱要原則

| 原則 | 結果 |
|---|---|
| 型別化記錄 | 每個公開物件皆為版本化且與綱要綁定 |
| 事件導向分類帳 | 獎勵會計衍生自可審計的歷史事件 |
| 資料分離 | 個人資料、彙總資料與鏈上摘要各自居於獨立層級 |
| 可版本化 API | 欄位新增/移除遵循向後相容規則 |
